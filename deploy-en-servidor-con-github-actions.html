Deploy en Servidor con GitHub Actions

GitHub Actions ofrece una manera poderosa y automatizada de construir, probar e implementar tu código directamente desde tu repositorio de GitHub. En este artículo, te guiaré paso a paso para configurar un flujo de trabajo de GitHub Actions que despliegue automáticamente tu repositorio a un servidor tras cada push a la rama principal.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Un repositorio en GitHub: El repositorio que deseas desplegar.

  • Un servidor accesible por SSH: Un servidor (virtual, dedicado, etc.) al que puedas acceder mediante SSH para desplegar tus archivos.

  • Credenciales de acceso al servidor: Necesitarás las credenciales SSH (clave privada o usuario/contraseña) para acceder a tu servidor de forma segura.

Paso 1: Configura los Secretos de GitHub

Para automatizar el despliegue de forma segura, utilizaremos los "Secretos" de GitHub para almacenar información sensible como las credenciales de acceso a tu servidor.

  1. Ve a la configuración de tu repositorio: En tu repositorio de GitHub, haz clic en la pestaña "Settings".

  2. Selecciona "Secrets and variables" y luego "Actions": En la barra lateral izquierda, busca y selecciona "Secrets and variables", y dentro de ese menú, haz clic en "Actions".

  3. Crea nuevos secretos: Haz clic en el botón "New repository secret" para crear los siguientes secretos. Reemplaza los valores de ejemplo con tus propias credenciales:

    • SSH_PRIVATE_KEY: Pega el contenido de tu clave privada SSH. Asegúrate de que esta clave privada tenga permisos para acceder a tu servidor.

    • SERVER_HOST: La dirección IP o nombre de dominio de tu servidor.

    • SERVER_USERNAME: El nombre de usuario para acceder a tu servidor.

    • SERVER_PORT: (Opcional): Si tu servidor SSH utiliza un puerto diferente al puerto 22, define este secreto.

    • TARGET_PATH: La ruta completa en tu servidor donde quieres desplegar los archivos de tu repositorio (ej. /var/www/mi-aplicacion)

Maneja tus secretos con cuidado. No los compartas públicamente y asegúrate de que tu clave privada SSH esté protegida.

Paso 2: Crea el Workflow de GitHub Actions

Ahora, crearemos el archivo de workflow que define los pasos para el despliegue.

  • Crea el directorio .github/workflows: Si no existe, crea un directorio llamado .github en la raíz de tu repositorio, y dentro de él, crea otro directorio llamado workflows.

  • Crea un archivo YAML para el workflow: Dentro del directorio workflows, crea un nuevo archivo con extensión .yml (ej. deploy.yml).

  • Define el workflow: Abre el archivo deploy.yml y pega el siguiente código. Asegúrate de leer y entender cada sección para personalizarlo según tus necesidades.

    name: Deploy to Server on: push: branches: - main jobs: deploy: runs-on: ubuntu-latest steps: - name: Checkout del código fuente uses: actions/checkout@v3 - name: Desplegar al servidor mediante SCP uses: appleboy/scp-action@master with: host: ${{ secrets.SERVER_HOST }} username: ${{ secrets.SERVER_USERNAME }} port: ${{ secrets.SERVER_PORT }} # (opcional) key: ${{ secrets.SSH_PRIVATE_KEY }} source: ./ target: ${{ secrets.TARGET_PATH }} strip_components: 1

    Explicación del Workflow (detallada)

    • name: Deploy to Server: Define el nombre del workflow, que aparecerá en la interfaz de GitHub Actions.

    • on: push: branches: - main: Especifica que este workflow se ejecutará automáticamente cada vez que haya un push a la rama main de tu repositorio. Puedes modificar las ramas o eventos de activación según tus necesidades.

    • jobs: deploy:: Define un trabajo llamado deploy. Los workflows pueden contener múltiples trabajos que se ejecutan en paralelo o secuencialmente.

    • runs-on: ubuntu-latest: Indica que este trabajo se ejecutará en un entorno virtual con Ubuntu.

    • steps:: Define una serie de pasos que se ejecutarán dentro del trabajo deploy.

      • actions/checkout@v3: Esta acción oficial de GitHub Actions obtiene una copia del código de tu repositorio en el entorno virtual. Es un paso esencial para poder desplegar tu código.

      • appleboy/scp-action@master: Esta es una acción de terceros que utiliza scp (Secure Copy) para copiar archivos de forma segura a un servidor remoto.

      • host, username, port, key: Estos parámetros utilizan los secretos que configuraste en el Paso 1 para establecer la conexión SSH con tu servidor.

      • source: ./: Indica que se copiarán todos los archivos desde el directorio raíz del repositorio (./).

      • target: ${{ secrets.TARGET_PATH }}: Define la ruta de destino en tu servidor donde se copiarán los archivos, utilizando el secreto TARGET_PATH

      • strip_components: 1: Este parámetro es útil si no quieres crear una subcarpeta adicional en el directorio de destino en tu servidor. Elimina el directorio raíz del repositorio al copiar los archivos.

Paso 2: Probar el despliegue

  1. Realiza un cambio en tu repositorio y haz push a la rama main:

    git add . && git commit -m "Test deploy" && git push origin main

  2. Ve a la pestaña Actions de tu repositorio en GitHub para ver el estado del despliegue.

  3. Si todo funciona, los archivos aparecerán en la ruta definida en TARGET_PATH.

Conclusión

Con este workflow, has automatizado el despliegue de tu aplicación en un servidor usando GitHub Actions y SCP. Este enfoque es ideal para proyectos pequeños o medianos, aunque para entornos más complejos, considera usar rsync o integraciones con Docker. Siempre protege tus secretos y verifica los permisos de SSH.
